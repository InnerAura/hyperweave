## The Problem

You want to visualize session data from a supported agent runtime -- tool usage, cost, token counts, and workflow stages -- as structured SVG artifacts.

## Parse a Session

First, parse the raw JSONL transcript into a telemetry contract. HyperWeave auto-detects Claude Code, Codex, and Antigravity transcripts from their JSONL shape:

```bash
hyperweave session parse path/to/session.jsonl
```

This outputs JSON with the full session analysis: tools, stages, corrections, cost, and profile data.

Common transcript locations include:

```bash
hyperweave session parse ~/.claude/projects/<project>/sessions/<session-id>.jsonl
hyperweave session parse ~/.codex/sessions/.../<session-id>.jsonl
hyperweave session parse ~/.gemini/antigravity/brain/.../<session-id>.jsonl
```

## Render a Receipt

Render a receipt card from a transcript:

```bash
hyperweave session receipt path/to/session.jsonl
```

Receipts are written to `.hyperweave/receipts/` by default, with filenames derived from the session timestamp and title or prompt. Override the output path with `-o`:

```bash
hyperweave session receipt transcript.jsonl -o my-receipt.svg
```

Use `--genome` when you want to pin a consistent telemetry skin:

```bash
hyperweave session receipt transcript.jsonl --genome voltage
```

The receipt includes a one-line summary on stderr:

```
Receipt: $0.42 · 12.5K tokens · 8m -> .hyperweave/receipts/2026-06-16-example.svg
```

## Render a Rhythm Strip

Rhythm strips show tool call density and classification over time:

```bash
hyperweave session strip transcript.jsonl -o rhythm.svg
```

## Using the Data Contract Directly

For custom rendering or integration, use the Python API:

```python
from hyperweave.telemetry.contract import build_contract

contract = build_contract("path/to/session.jsonl")
# Returns dict with: session, tools, stages, corrections, profile, cost
```

Then compose with the contract:

```python
from hyperweave.compose.engine import compose
from hyperweave.core.models import ComposeSpec

spec = ComposeSpec(type="receipt", telemetry_data=contract)
result = compose(spec)
# result.svg contains the receipt SVG
```

## Telemetry Frame Types

| Frame | Description |
|---|---|
| `receipt` | Session summary card with cost, tokens, duration, tools |
| `rhythm-strip` | Timeline showing tool call density and classification |

<Info>
  Session commands auto-detect the telemetry skin from the transcript runtime. Use `--genome` on CLI commands to pin a telemetry skin such as `voltage` when you want a consistent livery.
</Info>

## MCP Integration

```
hw_compose(
  type="receipt",
  telemetry_data={...contract dict...}

```

## Hook-Based Automation

For automatic receipt generation after every session, install hooks for supported runtimes:

```bash
hyperweave install-hook
```

You can also scope installation to one runtime:

```bash
hyperweave install-hook --runtime claude-code
hyperweave install-hook --runtime codex
hyperweave install-hook --runtime antigravity
```

To backfill Antigravity receipts from existing transcripts:

```bash
hyperweave session sync --runtime antigravity
```

See the [Install Session Telemetry](/tutorials/install-session-telemetry) tutorial for details.
